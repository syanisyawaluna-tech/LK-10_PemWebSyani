# Laporan Perbaikan Bug dan Refactoring (LK-11)

Laporan ini mendokumentasikan proses identifikasi bug, perbaikan, dan refactoring pada proyek Laravel **Sistem Peminjaman Barang Lab Riset** (LK-10) menggunakan bantuan AI Code Assistant.

## 1. AI Usage Log
**Tool:** Antigravity AI (Gemini 3.5 Flash)
**Task:** Analisis kode, Identifikasi Bug, dan Rekomendasi Refactoring.

**Aktivitas:**
1.  **Analisis Controller:** Mengidentifikasi adanya duplikasi kode pada logika validasi dan penggunaan metode pengambilan model yang masih manual (boilerplate).
2.  **Rekomendasi Struktur:** AI menyarankan penggunaan *Form Request* untuk memisahkan logika validasi dari Controller guna menjaga prinsip *Single Responsibility*.
3.  **Optimalisasi Route:** AI mendeteksi adanya rute login yang redundan dan menyarankan konsolidasi rute untuk meningkatkan keterbacaan kode (*maintainability*).
4.  **UX Improvement:** Memberikan saran untuk menambahkan pesan error kustom dalam bahasa Indonesia agar sesuai dengan target pengguna aplikasi.

## 2. Bug Fix Log

### Bug 1: Duplikasi Logika Validasi
*   **Masalah:** Aturan validasi (NIM, Email, Angkatan) ditulis dua kali di metode `store` dan `update`. Jika ada perubahan aturan (misal: panjang NIM), pengembang harus mengubah di dua tempat berbeda.
*   **Sebelum Perbaikan:** Validasi dilakukan manual di dalam `MahasiswaController` menggunakan `$request->validate([...])`.
*   **Sesudah Perbaikan:** Semua aturan validasi dipindahkan ke satu file pusat di `app/Http/Requests/MahasiswaRequest.php`.

### Bug 2: Penggunaan Data Tidak Tervalidasi (*Mass Assignment Risk*)
*   **Masalah:** Penggunaan `Mahasiswa::create($request->all())` berisiko jika ada kolom tambahan yang dikirim melalui form (meskipun sudah ada `$fillable`).
*   **Sebelum Perbaikan:** Menggunakan `$request->all()`.
*   **Sesudah Perbaikan:** Menggunakan `$request->validated()` yang hanya mengambil data yang benar-benar lolos aturan validasi.

### Bug 3: Redundansi Rute Login
*   **Masalah:** Terdapat tiga rute berbeda yang merujuk pada view login yang sama (`/`, `/auth/login`, `/login-page`). Hal ini membingungkan dan membuat pengelolaan rute menjadi tidak efisien.
*   **Sebelum Perbaikan:** Rute tersebar di 3 definisi berbeda di `web.php`.
*   **Sesudah Perbaikan:** Dikonsolidasi menjadi rute bernama `login` dan rute `/` dialihkan (*redirect*) ke rute tersebut.

## 3. Refactoring Log

### Refactor 1: Implementasi Form Request (`MahasiswaRequest`)
*   **Tujuan:** Memindahkan logika bisnis validasi keluar dari Controller.
*   **Perubahan:** Membuat file `MahasiswaRequest.php`. Controller kini lebih bersih (*slim*) karena hanya menerima data yang sudah bersih dari Request tersebut.
*   **Manfaat:** Kode lebih mudah dibaca, diuji, dan dipelihara.

### Refactor 2: Implementasi Route Model Binding
*   **Tujuan:** Menghapus kode manual `Mahasiswa::findOrFail($id)` di setiap metode CRUD.
*   **Perubahan:** 
    - Mengubah parameter metode `edit`, `update`, dan `destroy` dari `int $id` menjadi `Mahasiswa $mahasiswa`.
    - Laravel secara otomatis mencari model berdasarkan ID yang ada di URL.
*   **Manfaat:** Mengurangi baris kode (*boilerplate*) dan otomatis menangani error 404 jika data tidak ditemukan.

Melalui bantuan AI, kode proyek LK-10 berhasil ditingkatkan kualitasnya. Controller menjadi jauh lebih ringkas, logika validasi lebih aman dan terpusat, serta struktur rute aplikasi menjadi lebih rapi dan konsisten.
